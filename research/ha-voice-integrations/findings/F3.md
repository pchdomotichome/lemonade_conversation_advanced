# F3: Modern Voice Pipeline Patterns

## Findings

### [1] Home Assistant Assist pipeline is a three-stage architecture: STT → Conversation Agent (Intent) → TTS, configurable with interchangeable components
- quote: "For each component, you can choose from different options."
- url: https://www.home-assistant.io/voice_control/voice_remote_local_assistant/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [2] Speech-to-Phrase is a close-ended STT engine that transcribes commands in under 1 second on Raspberry Pi 4 or Home Assistant Green, launched February 2025
- quote: "Extremely fast transcription even on a Home Assistant Green or Raspberry Pi 4 (under one second)."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [3] Speech-to-Phrase auto-generates phrases and fine-tunes a model based on the user's Home Assistant devices, areas, and sentence triggers — fully locally and offline
- quote: "Speech-to-Phrase will automatically generate the phrases and fine-tune a model based on the devices, areas, and sentence triggers in your Home Assistant server - 100% locally and offline."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [4] Whisper remains available as open-ended STT but takes ~8s on Pi4 vs under 1s on Intel NUC — recommended only when paired with LLM agents
- quote: "On a Raspberry Pi 4, it takes around 8 seconds to process incoming voice commands. On an Intel NUC, it is done in under a second."
- url: https://www.home-assistant.io/voice_control/voice_remote_local_assistant/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [5] Piper is a fast local neural TTS engine optimized for Raspberry Pi 4, generating 1.6s of speech per second of processing
- quote: "On a Raspberry Pi, using medium quality models, it can generate 1.6s of voice in a second."
- url: https://www.home-assistant.io/voice_control/voice_remote_local_assistant/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [6] LLM-based conversation agents (OpenAI, Anthropic, Ollama, Google Generative AI) can control Home Assistant devices via the Assist API with entity exposure
- quote: "Controlling Home Assistant is done by providing the AI access to the Assist API of Home Assistant. You can control what devices and entities it can access from the exposed entities page."
- url: https://www.home-assistant.io/integrations/openai_conversation/
- source_type: primary
- published: 2026-07-01
- confidence: high

### [7] Streaming responses were added in HA 2025.3 for Ollama and OpenAI, allowing users to read replies as they're generated and execute commands before the full response completes
- quote: "In Home Assistant 2025.3 we're introducing support for LLMs to stream their response to the chat, allowing users to start reading while the response is being generated. A bonus side effect is that commands are now also faster: they will be executed as soon as they come in, without waiting for the rest of the message to be complete."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [8] Home Assistant supports Model Context Protocol (MCP) for bidirectional LLM integration — HA can both consume external MCP servers and expose its tools to other AI apps via MCP server integration
- quote: "Using the new Model Context Protocol integration, Home Assistant can integrate external MCP servers and make their tools available to LLMs that Home Assistant talks to... the new Model Context Protocol server integration, Home Assistant's LLM tools can be included in other AI apps, like the Claude desktop app."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [9] Conversation history is shared between the default (built-in) conversation agent and its LLM fallback, enabling follow-up questions that reference recent commands
- quote: "Now, the conversation history is shared with the LLM. The context allows you to ask the LLM for follow-up questions that refer to recent commands, regardless of whether they helped process the request."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [10] "Prefer handling commands locally" is a setting introduced in HA 2024.12 that routes simple commands to the built-in agent first, falling back to the LLM only for unrecognized input
- quote: "One major change was the new 'prefer handling commands locally' setting, which always attempts to run commands with the built-in conversation agent before it sends it off to an LLM."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [11] Scripts exposed to Assist become callable LLM tools (not entities), with a hard limit of 128 scripts/tools, each requiring a description and field descriptions capped at 128 characters
- quote: "Do not expose more than 128 scripts/tools in total. Exceeding this limit will cause the conversation engine to fail with a hard limit error inherited from the underlying LLM API."
- url: https://www.home-assistant.io/voice_control/exposing_scripts_to_llms/
- source_type: primary
- published: 2026-07-01
- confidence: high

### [12] The Wyoming protocol is the standard for linking voice pipeline components; Home Assistant 2025.3 added support for announcements to Wyoming satellites and microWakeWord as an alternative to openWakeWord
- quote: "Wyoming is our standard for linking together all the different parts needed to build a voice assistant... We're also adding a new microWakeWord add-on (the same wake word engine running on Voice PE!) that can be used as an alternative to openWakeWord."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [13] Custom sentences are defined via YAML in `config/custom_sentences/<language>/` and support template syntax with `{name}` and `{area}` lists matching HA entities/areas
- quote: "Create a `custom_sentences/<language>` directory in your Home Assistant `config` directory where `<language>` is the language code of your language."
- url: https://www.home-assistant.io/integrations/conversation/
- source_type: primary
- published: 2026-07-01
- confidence: high

### [14] Home Assistant Voice Preview Edition is official voice hardware that sold out in 23 minutes at launch in December 2024, with an open ecosystem supporting ESPHome-based satellites
- quote: "It sold out 23 minutes into our announcement - wow!... The Voice PE uses a wizard to help you configure your system and get started to use voice."
- url: https://www.home-assistant.io/blog/2025/02/13/voice-chapter-9-speech-to-phrase/
- source_type: primary
- published: 2025-02-13
- confidence: high

### [15] LLM conversation agents support "Assist mode" (device control) vs "No control" (chat only), with a max description limit of ~1024 characters per script
- quote: "The integration must be configured to allow home control, that is, Assist mode, not No control."
- url: https://www.home-assistant.io/voice_control/exposing_scripts_to_llms/
- source_type: primary
- published: 2026-07-01
- confidence: high

### [16] The `conversation.process` action is the programmatic API for sending transcribed text to a conversation agent — enabling custom integrations and automations to trigger voice processing
- quote: "You can either converse by pressing the microphone in the frontend... or by calling the `conversation.process` action with the transcribed text."
- url: https://www.home-assistant.io/integrations/conversation/
- source_type: primary
- published: 2026-07-01
- confidence: high
