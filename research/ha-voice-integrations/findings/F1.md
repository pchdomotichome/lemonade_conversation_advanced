# F1: Home Assistant Official Voice Pipeline Architecture

## Findings

### [1] Assist is the built-in voice assistant in Home Assistant, introduced in HA 2023.2, running as the `assist_pipeline` integration (3190 active installations as of 2026.7)
- quote: "Assist is the voice assistant built into Home Assistant. It lets you control your smart home with natural language, and it can run fully on your own hardware, so your voice commands stay private."
- url: https://www.home-assistant.io/voice_control/
- source_type: primary
- published: 2026-07-08 (current doc)
- confidence: high

### [2] The Voice pipeline is a 5-component architecture: Assist Pipeline integration (orchestrator), Speech-to-Text (STT), Conversation (intent processing), Intent (execution), and Text-to-Speech (TTS)
- quote: "The Assist Pipeline integration is responsible for turning the user's speech into text, get it processed, and turn the response into speech. The Conversation integration is responsible for processing user's text. The Intent integration is responsible for executing the intent and returning a response."
- url: https://developers.home-assistant.io/docs/voice/overview
- source_type: primary
- published: unknown
- confidence: high

### [3] Intent recognition uses hassil, a template-matching engine that matches user input against sentence templates with slots (e.g., `(turn | switch) on [the] {area} lights`)
- quote: "Home Assistant's intent recognition is powered by hassil. Hassil recognizes intents by matching the user input against sentence templates."
- url: https://developers.home-assistant.io/docs/voice/intent-recognition
- source_type: primary
- published: unknown
- confidence: high

### [4] The built-in Conversation integration provides a `conversation.process` action that accepts text input and routes it to conversation agents; it supports custom YAML sentence templates via `custom_sentences/<language>/` directories
- quote: "The Conversation integration allows you to converse with Home Assistant. You can either converse by pressing the microphone in the frontend or by calling the conversation.process action with the transcribed text."
- url: https://www.home-assistant.io/integrations/conversation/
- source_type: primary
- published: 2026-07-08 (current doc)
- confidence: high

### [5] The ConversationEntity class (derived from `homeassistant.components.conversation.ConversationEntity`) accepts `ConversationInput` with fields: text, context, conversation_id, language, continue_conversation; it returns `ConversationResult` with an IntentResponse
- quote: "A conversation entity is derived from the homeassistant.components.conversation.ConversationEntity. A ConversationInput object contains: text, context, conversation_id, language, continue_conversation."
- url: https://developers.home-assistant.io/docs/core/entity/conversation
- source_type: primary
- published: unknown
- confidence: high

### [6] There are 38+ supported built-in intents including HassTurnOn/Off, HassGetState, HassLightSet, HassClimateSetTemperature, HassStartTimer, HassMediaSearchAndPlay, HassVacuumStart, HassFanSetSpeed, etc. Intents are provided by their respective domain integrations
- quote: "HassTurnOn, HassTurnOff, HassGetState, HassNevermind, HassRespond, HassBroadcast, HassSetPosition, HassGetCurrentDate, HassGetCurrentTime, HassLightSet, HassClimateSetTemperature, HassStartTimer..."
- url: https://developers.home-assistant.io/docs/intent_builtin/
- source_type: primary
- published: unknown
- confidence: high

### [7] Local STT options are Speech-to-Phrase (close-ended, fast, <1s on RPi4) and Whisper (open-ended, ~8s on RPi4, <1s on Intel NUC); local TTS is Piper (neural TTS, 1.6s audio generated per 1s on RPi4)
- quote: "Speech-to-Phrase is a close-ended speech model... Extremely fast transcription even on a Home Assistant Green or Raspberry Pi 4 (under one second). Piper is a fast, local neural text-to-speech system that sounds great and is optimized for the Raspberry Pi 4."
- url: https://www.home-assistant.io/voice_control/voice_remote_local_assistant/
- source_type: primary
- published: unknown
- confidence: high

### [8] The STT entity interface requires implementations of `SpeechToTextEntity` with properties: supported_languages, supported_formats (wav/ogg), supported_codecs (pcm/opus), supported_bit_rates (8/16/24/32), supported_sample_rates, supported_channels (1/2)
- quote: "A speech-to-text (STT) entity allows other integrations or applications to stream speech data to the STT API and get text back."
- url: https://developers.home-assistant.io/docs/core/entity/stt
- source_type: primary
- published: unknown
- confidence: high

### [9] The TTS entity interface supports both 1-shot generation (`get_tts_audio`/`async_get_tts_audio`) and streaming generation via `async_stream_tts_audio(TTSAudioRequest) -> TTSAudioResponse` for LLM chunked text output
- quote: "Large language models generate text in chunks. The TTS service can be called with a stream of text messages, and the TTS service will return the audio in chunks."
- url: https://developers.home-assistant.io/docs/core/entity/tts
- source_type: primary
- published: unknown
- confidence: high

### [10] Voice Satellites (ESPHome-based ESP32 devices, Android/iOS apps, or Linux devices) detect wake words, capture speech, send it to HA, and play back responses; the recommended hardware is the Voice Preview Edition
- quote: "The ultimate goal is to make Voice Satellites. These are devices that can be placed anywhere in the house. Once it detects the hot word, it will capture the user's speech, send it to Home Assistant, and play the response back to the user."
- url: https://developers.home-assistant.io/docs/voice/overview
- source_type: primary
- published: unknown
- confidence: high

### [11] Custom sentences are defined via YAML in `config/custom_sentences/<language>/` directories, supporting slot-based matching with built-in entity/area lists (`{name}`, `{area}`), and can trigger either built-in intents or custom `intent_script` actions
- quote: "Create a custom_sentences/<language> directory... These YAML files are automatically merged, and may contain intents, lists, or expansion rules."
- url: https://www.home-assistant.io/integrations/conversation/
- source_type: primary
- published: 2026-07-08 (current doc)
- confidence: high

### [12] The Conversation integration also supports sentence triggers for automations with wildcard support (e.g., `play {album} by {artist}`) and conversation response script actions for dynamic TTS replies
- quote: "You may add your own sentences to the intent recognizer by either extending an existing intent or creating a new one."
- url: https://www.home-assistant.io/voice_control/custom_sentences/
- source_type: primary
- published: unknown
- confidence: high
